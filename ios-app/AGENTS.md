# iOS UI scope

Read the root guide and platform parity docs. Use generated `PFColorToken`, spacing, typography, component styles, and `PFSemanticIcon`; keep SF Symbols and SwiftUI behavior native. Do not edit generated files or replace accessibility identifiers. Reusable product components need previews for canonical scenarios, light/dark, and Dynamic Type. Verify with the affected tests and `xcodebuild` when practical.
