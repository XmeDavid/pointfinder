## High Priority — Open

### 1. `AssignmentResolver.resolve()` O(n) per-call regression in MonitoringService
**Flagged by**: Architect

The old `MonitoringService` pre-indexed assignments into `HashMap`s for O(1) lookup per base. The new `AssignmentResolver.resolve()` does a full linear scan of ALL assignments per call. For monitoring progress (T teams x B bases x A assignments), this is O(T*B*A) vs the old O(T*B).

**Fix**: Add a bulk `resolveAll()` method that builds the index once, or accept pre-indexed maps.

### 2. `findByBaseId` in `AssignmentRepository` has no JOIN FETCH — N+1
**Flagged by**: Architect

`PlayerService.submitAnswer()` and `buildCheckInResponse()` call `findByBaseId()`, a Spring Data derived query with no fetch joins. `AssignmentResolver.resolve()` then accesses `.getTeam()`, `.getBase()`, `.getChallenge()` — each triggering lazy loads.

**Fix**: Replace with JPQL + JOIN FETCH.

## Medium Priority — Open

### 3. WebSocket connection limit (200/game) exhaustible by single client
**Flagged by**: Sentinel

No per-principal limit. One authenticated client could open 200 connections and lock out all other players.

### 4. `ResetPasswordPage.tsx` has no client-side password length check
**Flagged by**: Scout

The reset password form submits without any length validation. Backend rejects it, but the raw error message is displayed.

### 5. "Game not active" overlay shows wrong message when game has ended
**Flagged by**: Scout

Both "setup" and "ended" states show "Please wait until your operator sets the game live" — misleading for ended games.

### 6. No haptic feedback on NFC check-in (iOS + Android)
**Flagged by**: Scout

The most exciting moment in the game (scanning an NFC tag) has no tactile feedback. Only QR scanning has haptics.

### 7. No "batch approve" for submissions
**Flagged by**: Operator

When 30+ submissions pile up during a live game, operators must review each individually.

### 8. `PlayerService` is a god service (14 dependencies, 554 lines, 4+ responsibilities)
**Flagged by**: Architect

Handles joining, check-ins, progress, submissions, push tokens, account deletion, locations, and notifications. Consider splitting.

### 9. Grafana password required may break existing deployments
**Flagged by**: Firefighter

Existing `.env` files without `GRAFANA_ADMIN_PASSWORD` will prevent the compose stack from starting. Document in deploy notes.
